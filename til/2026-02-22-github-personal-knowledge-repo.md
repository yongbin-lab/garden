# GitHub로 개인 지식 저장소 만들기

회사에서 배운 것, 읽은 책, 만든 플러그인 등 흩어져 있던 지식을 한 곳에 모으기 위해 GitHub 레포를 세팅했다.

## 구조

- **Public (`garden`)**: 공개 가능한 인사이트, TIL, 플러그인, 책 요약
- **Private (`vault`)**: 회사 업무 관련 학습, 회고, 공개 전 초안

## 왜 GitHub인가

- 마크다운 기반이라 가볍고, 어디서든 접근 가능
- 커밋 히스토리가 곧 학습 기록이 된다
- Public 레포 자체가 포트폴리오이자 퍼스널 브랜딩

## 배운 점

- Public/Private를 분리하면 회사 관련 민감한 내용도 부담 없이 기록할 수 있다
- 완벽한 글을 쓰려고 하지 말고, `til/`처럼 짧게라도 매일 기록하는 습관이 중요하다
- 초안은 `vault/drafts/`에서 작성하고, 정리되면 `garden`으로 옮기는 워크플로우가 자연스럽다
